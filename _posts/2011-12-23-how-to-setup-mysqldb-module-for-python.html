--- title: "How to setup MySQLdb Module for Python MySQL connection on Mac" layout: "post" permalink: "/2011/12/how-to-setup-mysqldb-module-for-python.html" ---
As Python doesn't provide a native module to connect to MySQL server (which is bad for web developer like me), we have to setup the connection module by ourselves. The best module is MySQLdb, which is developed by Andy Dustman. This project is hosted on sourceforge, you can find the project page at: http://sourceforge.net/projects/mysql-python/

However, this module is developed in for linux distribution, so it is a bit painful process to install this module on Mac. I googled around and found some articles claims that the easiest solution on Mac is to utilize complete solution like installing the whole Python/ Mysql stack through MacPort. I have to say that this is true if you have a empty box, but this is not the case for me. As I also develop PHP application, I have MySQL installed already.  So I struggled to find a solution to build and install MySQLdb module from source.

I note the process down here for my own reference and also hope it will be helpful for someone else.

Environment:    MacBook Pro Version 10.6.8

I have the following dependence already installed

1)  Python 2.6.1  64bit
2)  MySQL Server 5.1.60 64bit

Note:  MySQLDB module requires Python and MySQL server has same architecture , otherwise it may have comparability problem.

Step1)  Download MySQLdb module MySQL-python-1.2.3.tar.gz from http://sourceforge.net/projects/mysql-python/

Step2) uncompress the package and enter it
tar -xvf MySQL-python-1.2.3.tar.gz
cd MySQL-python-1.2.3

3) try to build the module

sudo python setup.py build

it will report you build error, you will notice one line of error:

_mysql.c:36:23: error: my_config.h: No such file or directory

this is caused by not setting mysql_config path correctly.

4) modify mysql_config path

Use your favorite text editor to open up file setup_posix.py
find text :  mysql_config.path = "mysql_config" 
 change this setting to your MySQL server mysql_config file location, in my case, it is:
/usr/local/mysql-5.1.60-osx10.6-x86_64/bin/mysql_config
save the change and quit

5) build and install the module again
sudo python setup.py build
sudo python setup.py install

This should work fine now and the MySQLdb module is installed successfully.

6) Verify module installation
in a shell, type python to enter python environment
>>>  import MySQLdb

This command should be executed without any complaint from intepreter
