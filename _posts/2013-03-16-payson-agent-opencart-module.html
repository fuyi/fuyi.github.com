This weekend, I have been working on integrating payson to a ecommerce website . (payson is a swedish leading online payment service provider).


There are 2 ways to integrate with payson payment gateway, one is through payson API, (http://api.payson.se/); the other one is through payson agent.
The problem with payson API approach is that payer is required to login to payson account in order to finish payment. this is not very convenient.
payson agent doesn't require payer to login and payment can be made by credit card and internet bank directly.

Taking user experience into consideration, I decide to use payson agent integration. the website is based on opencart. unfortunately, payson doesn't provide official payment module for opencart, so I created one by myself.

Suported features:

1)Payson Guarantee (Paysongaranti) Escrow service support
2) Multiple payment options: credit card only, internet bank only, deposit in payson account, and all of them.
3) English and Swedish supported.
4) Supported Currencies: Swedish Krona (SEK).

the code is on github: https://github.com/fuyi/payson-agent-opencart-module

If you want to see how it works live, you can visit the commerce site:  www.bycase.se
