My current rails project requires integration with Viagogo (online ticket marketplace for buying and selling live event tickets). besides they have a API (private instead of public), their are several technical barrier to use their API

1) they use old-fashioned OAuth1 protocol, not the latest OAuth2 protocol
2) they don't fully follow OAuth1 specification, which makes it impossible to use standard oauth gem for authentication and make  service call
3) unfortunately, they don't have a official client library, I can not find an unofficial one either.

So, I have to write the ruby client all by myself. I have finished the first version and push to github and rubygems.org.

This version is very basic, but allow you to avoid cumbersome authentication process and query events by free text

Github:  https://github.com/fuyi/viagogo-ruby-client
Rubygems  http://rubygems.org/gems/viagogo
